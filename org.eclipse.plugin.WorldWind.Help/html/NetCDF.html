NetCDF

NetCDF (network Common Data Form) is a set of software libraries and machine-independent data formats that support the creation, access, and sharing of array-oriented scientific data.

http://www.unidata.ucar.edu/software/netcdf/

World Wind Geo uses the NerCDF library for Java http://www.unidata.ucar.edu/software/netcdf-java/ to access scientific data in various formats including:

  • NetCDF-3 (classic and large format)
  • HDF5 (version 4.0 will be mostly complete, earlier versions are partial)
  • OPeNDAP client/server protocol
  • GRIB version 1 and version 2 (ongoing work on tables)
  • BUFR (limited - ongoing work)
  • GINI (GOES Ingest and NOAAPORT Interface ) image format
  • GEMPAK gridded data (as of version 2.23)
  • DMSP (Defense Meteorological Satellite Program)
  • NEXRAD Radar level 2 and level 3.
  • ATD's DORADE radar file format
  • ADDE client/server protocol (limited)

To access a NetCDF data set simply select "Open Dataset" from the file menu or point the built-in web browser to an OpeNDAP server for NetCDF. As soon as you click an hyperlink the dataset will be opened in the NetCDF View.

Here is a list of sample servers
  • COLA Climate Modeling Data Sets: http://dataportal.ucar.edu:9191/dods
  • COLA Weather and Climate Data Sets: http://monsoondata.org:9090/dods
  • US Global Ocean Data Assimilation Experiment (GODAE): http://www.usgodae.org/dods/GDS
Note:Currently data grids must:
  1. have a 1D X and Y coordinate axes.
  2. be lat/lon or Lambert Conformal Projection.
  3. be equally spaced.
